### A01.1 — Networking + Persistence (TDD ports of `Networking/*`, `Persistence/*`)
- `HttpMethod`, `ApiError` (sealed: Http/Transport/Decoding/NotImplemented +
  `fromResponse`/`messageFromBody` body-error extraction), `ApiClientConfig`
  (header merge + `X-Client-Platform: android` + 30s default), `ApiTrafficLogger`
  (OFF/SUMMARY/BODIES, `/auth/` redaction, injectable sink), `AuthTokenProvider`
  /`MutableTokenProvider`, `EmptyResponse`.
- `ApiClient` interface — single mockable `request(method, path, jsonBody,
  deserializer)` seam + reified `get/post/put/patch/delete` sugar (`ApiJson`).
- `OkHttpApiClient` — bearer injection, JSON bodies, `401 → ApiEvent.TOKEN_EXPIRED`,
  non-2xx → mapped `ApiError`, `EmptyResponse` short-circuit. The one OkHttp site.
- `TokenStore` interface + `InMemoryTokenStore` (Liskov twin) +
  `EncryptedTokenStore` (EncryptedSharedPreferences / Keystore — port of
  `KeychainTokenStore`).
- Tests: `ApiErrorTest`, `ApiClientConfigTest`, `OkHttpApiClientTest`
  (MockWebServer — decode, bearer, 401-emits-and-throws, non-2xx, malformed,
  EmptyResponse), `TokenStoreContractTest` (abstract Liskov contract) +
  `InMemoryTokenStoreTest`. `EncryptedTokenStoreInstrumentedTest` in **androidTest**
  (real Keystore; runs under `connectedCheck`, not JVM `check`).

## Validation

**The build gate has NOT been run.** This sandbox has **JDK 26** (Gradle 8.13 +
AGP 8.7 require JDK 17) and **no Android SDK** (`ANDROID_HOME` unset). Per the
owner's decision (2026-06-19), validation is delegated to Android Studio / the
`android-ci` workflow. To turn it green:

```bash
# JDK 17 (e.g. Temurin) as JAVA_HOME; Android SDK via local.properties / ANDROID_HOME
./gradlew check                 # ktlint + detekt + JUnit5 unit tests + lint
./gradlew dependencyBoundaryCheck
./gradlew :core:connectedCheck  # EncryptedTokenStore instrumented test (device/emulator)
```

## Known risks to watch on first real run (self-review, unvalidated)

- detekt `ThrowsCount` may flag `OkHttpApiClient.request` (3 throw sites) — bump
  the threshold in `detekt.yml` or extract if it fires.
- `EncryptedSharedPreferences` under Robolectric is unreliable → its contract is
  an **instrumented** test by design (documented divergence from the A01.1 plan,
  which named Robolectric).
- `MockWebServer` pulls JUnit4 onto the test classpath transitively; harmless
  under the JUnit5 platform (no vintage engine wired).

## Divergences from the iOS source (intentional)
- `APITrafficLogger` (os.Logger) → `ApiTrafficLogger` (android.util.Log + sink).
- Keychain → EncryptedSharedPreferences. `Data` → `ByteArray` (user blob).
- iOS `upload`/`getData` multipart not ported yet (no A01.x caller — added when
  a consumer needs it; NO OVERENGINEERING). `getData` default throws as on iOS.
