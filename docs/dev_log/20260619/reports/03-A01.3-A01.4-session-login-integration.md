## A01.3 — Session + Login UI

TDD port of `vbwd-ios-core` `Session/*`, `UI/Login/*`, `UI/RootRouter`,
`UI/RootView`. This turns `:core` into a **Compose + Hilt** module (the port of
the iOS `VBWDCore` hosting the shared UI).

### Source (`core/src/main/kotlin/com/vbwd/core/`)
- `session/AuthState.kt` — sealed interface `SignedOut | Authenticating |
  Authenticated(user) | Error(message)` (exhaustive `when`, no `else`).
- `session/AuthSession.kt` — `StateFlow<AuthState>` state machine over
  `AuthService` only (ISP/DIP). `start()` (restore), `signIn(Credentials)`,
  `signOut()`, `isAuthenticated`/`currentUser`/`accessToken`. **No EventBus** —
  deferred to A02 (no premature event layer); only `ApiError` is caught so
  `CancellationException` propagates.
- `ui/RootRouter.kt` — pure `AuthState → RootRoute` (`Login | Loading |
  Authenticated(user)`); `SignedOut`/`Error` → login (banner stays on login).
- `ui/RootView.kt` — Compose shell: restores once via `LaunchedEffect`, routes
  via `RootRouter`; the authenticated branch is a **slot** so `:core` stays free
  of the host shell (dashboard/plugin host supplied later).
- `ui/login/LoginViewModel.kt` — `@HiltViewModel` over `AuthSession`; `UiState`
  (email/password/isLoading/errorMessage + derived `canSubmit`); `submit()`
  guards incomplete input, signs in, surfaces the `Error` message.
- `ui/login/LoginScreen.kt` — thin Compose form; test tags mirror the iOS
  accessibility ids (`login_email_field` / `_password_field` / `_error_label` /
  `_submit_button`); `dp` literals hoisted to named vals (detekt MagicNumber).

### Tests
- `session/AuthSessionTest.kt` (Turbine + coroutines-test) — `SignedOut →
  Authenticating → Authenticated` (a gated fake holds `login` open so the
  transient state is observable), `→ Error` on failure, `signOut → SignedOut`,
  restore paths, `accessToken` delegation.
- `ui/RootRouterTest.kt` — every `AuthState` maps to the right `RootRoute`.
- `ui/login/LoginViewModelTest.kt` — `canSubmit` gating, guarded empty submit,
  credentials forwarded + loading cleared on success, error surfaced on failure
  (test Main dispatcher).
- **androidTest** (`connectedCheck`, JUnit4 + Compose rule, per the report-01
  instrumented-test precedent): `ui/login/LoginScreenTest.kt` (fields render,
  submit disabled-when-empty / enabled-when-filled, error label on failure) and
  `ui/RootViewTest.kt` (A01.4 boot: cold start → login slot; persisted session →
  authenticated slot).

## A01.4 — Integration

- `core/config/AppConfig.kt` — loads `api_base_url` from the `vbwd_config.json`
  asset; folds the iOS `load()?? defaultBaseURL` into one non-null call.
  Default `http://10.0.2.2:5000/api/v1` (emulator → host localhost).
- `app/di/CoreModule.kt` — the composition root (port of `SDKContainer`): the
  **only** place concrete adapters are named. Singletons for `AppConfig`,
  `ApiClientConfig`, `ApiClient` (`OkHttpApiClient`), `TokenStore`
  (`EncryptedTokenStore`), `AuthService`, `ProfileService`, an app
  `CoroutineScope`, and `AuthSession`. The `AuthSession` provider wires the
  **401 → auto sign-out** callback seam (web axios-interceptor parity; only when
  authenticated, so a failed login doesn't trigger it).
- `app/MainActivity.kt` — field-injects the `AuthSession` singleton, hands it to
  `AppRoot`.
- `app/AppRoot.kt` — `RootView` with `LoginScreen` (via `hiltViewModel()`) and an
  `AuthenticatedRootPlaceholder` (real shell in A02).
