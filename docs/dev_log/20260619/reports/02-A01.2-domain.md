## What landed

TDD port of `vbwd-ios-core` `Domain/*` (the A01.2 scope only — auth/profile;
store/cart left for A04). All under `core/src/main/kotlin/com/vbwd/core/domain/`.

### Models (`@Serializable`, field-for-field with the iOS Codable shapes)
- `AuthUser.kt` — `AccessLevel` + `AuthUser` with snake_case `@SerialName`
  (`is_admin`, `access_levels`, `user_access_levels`, `user_permissions`).
- `Credentials.kt` — `{email,password}`.
- `LoginResponse.kt` — `{success,token,refresh_token,user,user_id,error}`.
- `UserProfile.kt` — `UserProfile` (10 detail fields) + `ProfileResponse`,
  `ProfileResponseUser`, `UpdateDetailsResponse`, `ChangePasswordResponse`.
  Backend-null detail fields decode to `""` via the codec (see codec note).
- `DashboardModels.kt` — `Invoice`, `InvoiceLineItem`, `TokenTransaction`,
  internal response wrappers, `DashboardEndpoints`.

### Services (interface + default impl; DIP — depend on `ApiClient`/`TokenStore`)
- `AuthService` + `DefaultAuthService` — `login` (persists token/refresh/user
  blob only after a valid response; success=false/empty-token → `ApiError.Http(401)`),
  `logout` (best-effort server call, always clears locally), `restore`
  (token+user blob → user, else null), `fetchProfile`, `refreshAccessToken`
  (→ `ApiError.NotImplemented`, deferred to Sprint 02), `currentToken`.
- `ProfileService` + `DefaultProfileService` — `fetchProfile`/`updateDetails`
  (unwrap `details`, else `ApiError.Decoding`), `changePassword`.

### PermissionEvaluator
- `PermissionEvaluator.kt` — exact port of the web `auth.ts` rules: `*` grants
  all, exact match, `ns.*` prefix wildcard. Takes `List<String>` (ISP), single
  rule definition (DRY).

### Endpoints (overridable data classes, OCP)
- `AuthEndpoints` (`/auth/login|logout|refresh`, `profile=/user/profile`).
- `ProfileEndpoints` (`/user/profile`, `/user/details`, `/user/change-password`).

### Codec note (the one A01.1 touch)
- Added `coerceInputValues = true` to `ApiJson` so a backend `null` on a non-null
  field-with-default coerces to the default — the port of the iOS
  `decodeIfPresent(...) ?? ""` for `UserProfile`. Malformed JSON still throws
  `Decoding` (verified rule unchanged; the existing networking tests are
  unaffected — coercion only applies to declared defaults, not bad syntax).

### Tests (`core/src/test/kotlin/com/vbwd/core/domain/`)
- `PermissionEvaluatorTest` — parametrised `(granted, required, expected)` table
  mirrored from the iOS suite (superuser `*`, exact, `ns.*` prefix incl. the
  bare-`ns` non-match, deny, empty) + `hasAny` cases.
- `DomainModelJsonTest` — round-trips `AuthUser`/`LoginResponse` against the
  `Fixtures/*.json` copied into `core/src/test/resources/fixtures/`
  (`auth_user`, `login_success`, `login_error`) + the `UserProfile` null→""
  coercion.
- `DefaultAuthServiceTest` (MockK `ApiClient` via the `request` seam +
  `InMemoryTokenStore`) — login persists+returns; success=false → Http 401 and
  **nothing persisted**; transport error propagates without persisting; restore
  paths (present/no-token/no-blob); logout clears even on server failure;
  refresh → NotImplemented; currentToken; fetchProfile delegation.
- `DefaultProfileServiceTest` — fetch/update unwrap details else Decoding;
  changePassword posts to the endpoint.

## Divergences from the iOS source (intentional)
- iOS `AuthService` protocol-extension back-compat default for `currentToken`
  → Kotlin interface default method (`= null`).
- iOS `JSONEncoder`/`JSONDecoder` for the stored user blob → `ApiJson` +
  `AuthUser.serializer()` round-trip through `ByteArray`.
- `UserProfile` custom `init(from:)` (null→"") → declarative codec
  `coerceInputValues` (same observable behaviour, no hand-written decoder).
